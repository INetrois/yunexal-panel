Unsafe storage override (at your own risk)

Disables storage safety validation (critical mount/root/ext4-no-prjquota checks) for custom paths and migrations. The dropdown shows all mounted disks; blocked entries include the reason. This can break the host or corrupt system data.

Disk filesystem conversion

Formats a selected partition to ext4. This erases all data on that partition. After mounting, enable prjquota so per-container disk limits are enforced.

Storage Architecture
System partition — host OS and panel data
Keep system files separate from container payloads
Container partition — ext4 + prjquota
/var/lib/docker and volume data with enforced disk limits
Storage driver: overlay2
Per-container quota is backed by ext4 project quotas
Required mount option: prjquota
Without it, disk_limit policy is blocked or unsafe-only

ext4 + prjquota policy reference

Check Expected value
Filesystem type ext4 on target mount
Mount options Contains prjquota (or prjjquota)
Runtime behavior disk_limit applies via ext4 project quota IDs
Provisioning Use panel disk formatter (ext4 only), then remount with prjquota
Operational note Use storage migration to move existing server data onto ext4 + prjquota targets
